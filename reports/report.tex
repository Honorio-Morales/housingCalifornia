\section{Resumen Ejecutivo}

Este informe presenta el desarrollo completo de un sistema predictivo utilizando dos modelos de regresión: 
\begin{enumerate}
  \item \textbf{Regresión Lineal Múltiple}: modelo baseline simple.
  \item \textbf{Regresión Polinomial}: modelos de grado 2 y 3 para capturar relaciones no-lineales.
\end{enumerate}

Se utilizó el dataset \textit{California Housing} para predecir el valor medio de viviendas en distritos de California. 
El proyecto sigue la metodología CRISP-DM (Comprensión → Modelado → Evaluación → Despliegue) 
y entrega tanto un backend API (FastAPI) desplegado en Render como una interfaz web interactiva (HTML + Streamlit).

\textbf{Resultado principal:} El modelo de Regresión Polinomial de grado 2 obtiene el mejor desempeño, 
con $R^2 = 0.6070$ y RMSE $= 0.7176$ (unidades de 100k USD).

\section{Conceptos Teóricos Fundamentales}

\subsection{Regresión Lineal Múltiple}

La regresión lineal múltiple modela la relación entre una variable objetivo $y$ y múltiples variables predictoras $\mathbf{X}$ mediante:

\begin{equation}
y = \beta_0 + \beta_1 x_1 + \beta_2 x_2 + \cdots + \beta_p x_p + \epsilon
\end{equation}

donde:
\begin{itemize}
  \item $y$ es la variable objetivo (MedHouseVal en nuestro caso).
  \item $x_1, \ldots, x_p$ son las variables predictoras (MedInc, HouseAge, AveRooms, etc.).
  \item $\beta_0, \beta_1, \ldots, \beta_p$ son los coeficientes a estimar.
  \item $\epsilon$ es el término de error.
\end{itemize}

El objetivo es minimizar la suma de cuadrados de los errores (suma de cuadrados de residuos):

\begin{equation}
\text{SSE} = \sum_{i=1}^{n} (y_i - \hat{y}_i)^2
\end{equation}

\subsection{Regresión Polinomial}

La regresión polinomial extiende el modelo lineal al incluir términos polinómicos y de interacción. 
Para polinomios de grado $d$, el modelo se transforma según:

\begin{equation}
\mathbf{X}_{\text{poly}} = \left[ 1, x_1, x_2, \ldots, x_1 x_2, x_1^2, x_2^2, \ldots, x_p^d \right]
\end{equation}

Luego se aplica regresión lineal sobre el espacio expandido $\mathbf{X}_{\text{poly}}$. 
Esto permite capturar relaciones no-lineales entre características y el target, 
a costa de mayor riesgo de overfitting con grados muy altos.

\subsection{Métricas de Evaluación}

\subsubsection{Coeficiente de Determinación ($R^2$)}

Mide qué porcentaje de la varianza de $y$ es explicado por el modelo:

\begin{equation}
R^2 = 1 - \frac{\text{SSE}}{\text{SST}} = 1 - \frac{\sum (y_i - \hat{y}_i)^2}{\sum (y_i - \bar{y})^2}
\end{equation}

Rango: $[0, 1]$. Un $R^2$ cercano a 1 indica buen ajuste.

\subsubsection{Error Cuadrático Medio (RMSE)}

Es la raíz cuadrada del error cuadrático medio, en las mismas unidades que $y$:

\begin{equation}
\text{RMSE} = \sqrt{\frac{1}{n} \sum_{i=1}^{n} (y_i - \hat{y}_i)^2}
\end{equation}

Permite interpretar el error típico en términos prácticos.

\subsubsection{Análisis de Residuos}

Los residuos $\hat{\epsilon}_i = y_i - \hat{y}_i$ deben distribuirse aleatoriamente alrededor de 0 
sin patrones sistemáticos. Si hay patrones, indica violación de supuestos (linealidad, homocedasticidad).

\section{Descripción del Dataset}

\subsection{California Housing}

El dataset contiene información de 20,640 distritos de California, con 8 características originales:

\begin{itemize}
  \item \textbf{MedInc}: Ingreso mediano del distrito (en decenas de miles de USD).
  \item \textbf{HouseAge}: Edad media de las viviendas (en años), capped a 52.
  \item \textbf{AveRooms}: Número promedio de habitaciones por vivienda.
  \item \textbf{AveBedrms}: Número promedio de dormitorios por vivienda.
  \item \textbf{Population}: Población del distrito.
  \item \textbf{AveOccup}: Ocupación promedio (personas por vivienda).
  \item \textbf{Latitude}: Latitud del distrito.
  \item \textbf{Longitude}: Longitud del distrito.
  \item \textbf{MedHouseVal} (TARGET): Valor medio de la vivienda en el distrito (en cientos de miles de USD).
\end{itemize}

\textbf{Nota:} En este proyecto utilizamos 7 features (excluimos AveOccup) para mantener simplicidad.

\section{Metodología: CRISP-DM}

Nuestro proyecto sigue las fases de CRISP-DM:

\subsection{I. Comprensión de Datos}
\begin{itemize}
  \item Carga del dataset: 20,640 registros, 8 features + 1 target.
  \item Análisis de correlaciones y distribuciones.
  \item Identificación de variables clave.
\end{itemize}

\subsection{II. Preparación y Modelado}
\begin{itemize}
  \item Split train/test: 80/20 con seed=42 para reproducibilidad.
  \item Entrenamiento de tres modelos:
    \begin{enumerate}
      \item Regresión Lineal Múltiple (baseline).
      \item Regresión Polinomial de grado 2.
      \item Regresión Polinomial de grado 3.
    \end{enumerate}
\end{itemize}

\subsection{III. Evaluación}
\begin{itemize}
  \item Cálculo de $R^2$ y RMSE en el conjunto de prueba.
  \item Análisis de residuos para detectar patrones.
  \item Comparativa de modelos.
\end{itemize}

\subsection{IV. Despliegue}
\begin{itemize}
  \item API REST (FastAPI) en Render.
  \item Interfaz HTML interactiva.
  \item Aplicación Streamlit en Hugging Face Spaces (opcional).
\end{itemize}

\subsection{Comparativa de Métricas}

Regresión Polinomial (grado 3) & 0.5393 & 0.7770 \\
\hline
\end{tabular}
\caption{Desempeño de los tres modelos evaluados. El modelo polinomial de grado 2 presenta el mejor balance.}
\end{table}

\subsection{Interpretación}

\begin{itemize}
  \item \textbf{Modelo de referencia (Lineal):} Explica el 57.51\% de la varianza. Error típico de $\pm 0.7462$ en unidades de 100k USD (≈ \$74,620).
  
  \item \textbf{Polinomio grado 2 (MEJOR):} Explica el 60.70\% de la varianza, mejorando un 3.2\% respecto a lineal. RMSE desciende a 0.7176 (≈ \$71,760). 
    Las interacciones y términos cuadráticos capturan mejor la no-linealidad observada en scatter plots.
  
  \item \textbf{Polinomio grado 3:} Explica menos (53.93\%) y tiene RMSE más alto (0.7770). 
    Indicativo de \textbf{overfitting}: el modelo se ajusta demasiado al ruido en entrenamiento
    y generaliza peor en nuevos datos. Con 35 features (7 originales + interacciones + cúbicos), 
    la maldición de la dimensionalidad juega en contra.
\end{itemize}

\section{Justificación del Modelo Seleccionado}

Se selecciona \textbf{Regresión Polinomial de grado 2} por:

\begin{enumerate}
  \item \textbf{Mejor rendimiento}: $R^2 = 0.6070$ y RMSE $= 0.7176$, superiores a lineal.
  
  \item \textbf{Evita overfitting}: A diferencia de grado 3, mantiene un equilibrio entre complejidad y generalización.
  
  \item \textbf{Interpretabilidad}: 28 features (vs. 35 en grado 3) aún es manejable.
  
  \item \textbf{Justificación teórica}: El EDA mostró relaciones no-lineales; los términos cuadráticos e interacciones 
    son suficientes para capturar esas dinámicas.
\end{enumerate}

\section{Despliegue e Implementación}

\subsection{Arquitectura del Sistema}

El proyecto se estructura en tres componentes:

\subsubsection{1. Backend API (FastAPI)}
\begin{itemize}
  \item Framework: FastAPI + Uvicorn.
  \item Endpoint: \texttt{POST /predict} recibe JSON con las 7 features.
  \item Respuesta: predicción del modelo polinomial.
  \item Hosting: Render (https://housing-california-api.onrender.com).
  \item Despliegue: Docker, build automático desde Dockerfile.api en GitHub.
\end{itemize}

\subsubsection{2. Frontend HTML}
\begin{itemize}
  \item Formulario interactivo centrado, con campos para todas las features.
  \item Botón ``Predecir'': envía datos a la API y muestra resultado.
  \item Botón ``Simular'': genera valores aleatorios para explorar casos distintos.
  \item Resultado: muestra predicción en formato decimal y equivalente en USD (ej: 0.49 = \$49,000).
  \item Archivo: \texttt{web/index.html} (disponible en GitHub).
\end{itemize}

\subsubsection{3. Frontend Streamlit (Opcional)}
\begin{itemize}
  \item App interactiva: sliders para cada feature.
  \item Llamadas a la API remota en Render.
  \item Despliegue: Hugging Face Spaces.
\end{itemize}

\subsection{Flujo de Predicción}

\begin{lstlisting}[language=bash]
# Usuario ingresa datos en formulario HTML/Streamlit
Input: {MedInc: 3.0, HouseAge: 30, ...}

# Request POST a la API
curl -X POST https://housing-california-api.onrender.com/predict \
  -H "Content-Type: application/json" \
  -d '{"MedInc": 3.0, "HouseAge": 30, ...}'

# API:
#   1. Recibe JSON
#   2. Carga transformador PolynomialFeatures (grado 2)
#   3. Transforma input de 7 a 28 features
#   4. Predice con modelo pre-entrenado
#   5. Devuelve resultado JSON

Response: {"prediction": 0.4940}
\end{lstlisting}

\subsection{Persistencia de Modelos}

Los modelos se entrenan una sola vez localmente y se serializan:
\begin{itemize}
  \item \texttt{models/model\_linear.joblib}
  \item \texttt{models/model\_poly\_2.joblib}
  \item \texttt{models/poly\_2\_transform.joblib} (PolynomialFeatures)
  \item \texttt{models/model\_poly\_3.joblib}
  \item \texttt{models/poly\_3\_transform.joblib}
\end{itemize}

Estos archivos se incluyen en el repositorio GitHub y se copian al contenedor Docker durante el build.
